The modern web is broken. Pages load faster than ever but weigh more than ever. JavaScript bundles grow larger while core web vitals get worse. The paradox is real: we have more powerful tools than ever, and our pages are slower.

Static site generators offer a different path.

## The Static Advantage

Static sites are fast by default. No server-side rendering means no database queries, no API calls, no runtime computation. The page you see is the page that loads.

But speed is only part of the story. Static sites are also:

- **Secure** — No database to exploit, no server to compromise.
- **Reliable** — If it builds, it works. No runtime failures.
- **Cheap** — Hosted on CDNs, static sites cost pennies even at scale.
- **Accessible** — Clean HTML, no JavaScript dependency for core content.

## The Trade-offs

Static sites aren't for everything. Dynamic features — user accounts, real-time updates, complex forms — require additional infrastructure. But for content-focused sites (blogs, documentation, portfolios), static is often the better choice.

## The Philosophy

Choosing a static site generator isn't just a technical decision. It's a philosophical one. It says: content matters more than interactivity. Performance matters more than features. Simplicity matters more than complexity.

In an era of web bloat, that's a radical stance.
